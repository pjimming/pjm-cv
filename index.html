RLock | A Redis distributed lock developed based on Golang

Personal DevelopmentðŸ™‹

Under the distributed system, resources are protected by using locks, and due to the expiration mechanism of the Redis database, the possibility of deadlocks is largely avoided. At the same time, the lock implements functions such as reentrancy, blocking application, and asynchronous lease renewal. In the case of multiple Redis nodes, the RedLock algorithm is implemented to balance high availability and data consistency.

  • During lock, unlock and lease renewal operations, the atomicity of operations is satisfied by executing Lua scripts.
  • The Hash data structure is used to support the reentrancy of the lock. If the node resource that acquires the lock applies for the lock again, it will not be blocked.
  • When requesting a lock in a blocking manner, if the tryLock fails, the lock will be continuously applied for in a spinning manner until the lock is successfully applied or the block times out.
  • Use the watchdog to realize the asynchronous renewal of the Redis lock, and use the watchdog's running flag to ensure that the watchdog will not start repeatedly when the lock is re-entered. Pass the end command through the context to realize the termination and renewal operation of the watchdog.
  • Implement the RedLock algorithm to ease the contradiction between high availability and data consistency in the case of multiple Redis nodes.
